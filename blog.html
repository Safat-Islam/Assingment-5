What is event handling in the context of the DOM?

Event handling refers to the process of responding to user interactions with a web page, such as clicks, key presses, or mouse movements. In the DOM, you can attach event listeners to elements using methods like element.addEventListener("event", function), allowing you to define custom actions when an event occurs.

What is the difference between innerHTML and textContent?

innerHTML retrieves or sets the HTML content of an element, allowing you to include HTML tags and formatting. textContent, on the other hand, retrieves or sets the plain text content of an element, stripping out any HTML tags. Using textContent is often safer for preventing XSS attacks, as it treats the content as plain text.

How can you access an element in the DOM using JavaScript?
  • document.getElementById("elementId") for accessing an element by its ID.
  • document.getElementsByClassName("className") for elements with a specific class.
  • document.querySelector("selector") for a more versatile way to select elements using CSS selectors.
What are JavaScript closures?

A closure is a function that retains access to its lexical scope, even when the function is executed outside that scope. This means that a closure can remember the environment in which it was created. Closures are often used to create private variables or to maintain state in asynchronous programming.

How do you handle errors in JavaScript?

Errors in JavaScript can be handled using try...catch blocks. Code that may throw an error is placed inside the try block, and if an error occurs, the control is passed to the catch block where you can define how to handle the error. Optionally, a finally block can be added, which will execute regardless of whether an error occurred.
